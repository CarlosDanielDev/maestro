# Maestro Claude Code Template

This `.claude/` directory was generated by `maestro init`. It provides a complete Claude Code orchestration system with:

- **3 modes**: Vibe Coding (simple), Subagents Orchestrator (complex), Training (agent config)
- **5 agents**: Architect, QA, Security, Docs, Master Planner
- **TDD enforcement**: Tests are always written before implementation
- **API contract validation**: Ensures client models match backend schemas
- **Slash commands**: `/implement`, `/validate-contracts`

## Setup

1. Copy the `.claude/` directory into your project root
2. Edit `.claude/CLAUDE.md` — customize the **Project Technology Stack** section
3. Edit `.claude/skills/project-patterns/SKILL.md` — add your project's patterns
4. Optionally add MCP servers to `.claude/settings.json`

## Structure

```
.claude/
├── CLAUDE.md                          # Main orchestrator config (customize tech stack)
├── settings.json                      # MCP servers, hooks, env
├── agents/
│   ├── subagent-architect.md          # Architecture design (consultive)
│   ├── subagent-qa.md                 # Test design / QA (consultive)
│   ├── subagent-security-analyst.md   # Security review (consultive)
│   ├── subagent-docs-analyst.md       # Documentation (CAN write .md)
│   └── subagent-master-planner.md     # System design / ADRs (consultive)
├── commands/
│   ├── implement.md                   # /implement #123
│   └── validate-contracts.md          # /validate-contracts
├── skills/
│   ├── project-patterns/SKILL.md      # YOUR patterns (customize this!)
│   ├── api-contract-validation/SKILL.md
│   └── security-patterns/SKILL.md
└── hooks/
    └── README.md
```

## Customization

The key file to customize is `.claude/skills/project-patterns/SKILL.md` — this is where you document your project's architecture, conventions, and anti-patterns. The agents consult this file before making recommendations.
