## Highlights

Team orchestration ships. Define a team of agents, hand them a milestone, and maestro coordinates the work — scheduling issues by dependency level, dispatching them to the right agent, and surfacing live progress in the TUI graph view. The agent graph is now on by default; legacy state files migrate automatically on first run.

## Added

- Multi-agent orchestration foundation: L1 subagent dispatch with a cost-estimate formula, L2 per-issue orchestrator state machines, L3 cross-issue scheduler, team wizard TUI, and `maestro team {list,new,launch,manage,explain}` CLI subcommands with built-in preset docs under `docs/teams/` (#660, #661, #662, #663, #664, #665).
- `views.agent_graph_enabled = true` by default, with a TUI graph view of running sessions and an animated empty state when no sessions are active (#710, #692).
- CI error review-and-fix popup in the TUI (#695).

## Fixed

- Agent Graph no longer freezes after session completion (#694).

## Migration

State-store schema bumped from version 0 to 1. Legacy state files migrate automatically on first load — run `maestro` once after upgrade; no manual action required.

`agent_graph_enabled` is backfilled into older `maestro.toml` files on startup. The default value preserves prior behavior for users who never set the key.

New Cargo dependency: `directories = "5"` — resolves user-config paths under the orchestration foundation. Relevant for restricted-network and air-gapped builds.

## What's next

v0.28.0 ships the templates layer — a single canonical source for slash commands, agent rules, and subagent lists across every supported provider.
